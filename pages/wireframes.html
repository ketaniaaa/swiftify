Updated Wireframes

Home Page

  1. 1. Header

    The header is similar to the Spotify website as It is a dark band across the top with the logo and name of the site aligned to the let. The header contains the two fonts used throughout the website. Questrial is used for the name of the website and used throughout the site for headings and titles. The tab text font family is Work Sans, a sans-serif font which is recommended by the Spotify Design Guidelines for secondary text.

  2. 2. Banner

    the banner will display the purpose of the website and the use of the API. There is an image of a line chart which will be replaced with a site-related data image as decoration. Finally, there is a button at the bottom of the banner which is in a bold colour to draw the user's attention to it. This button will be a link to the data artwork that is being advertised.

  3. 3. Call to Action

    the call to action will give a short explanation as to why this Idea Is interesting or why a user might want to use a site of this nature The four circular bars are placeholders for images of CDs as this website is music themed. This use of circular images are used throughout the website to simulate the form of CDs and vinyls. Under each will be a short explanation of the different data visualizations that I will create throughout the semester for the user to be able to understand the fully purpose of the website upon immediate entry of the site.

  4. 4. Footer

    The footer remains the same across all pages of the website and will be a reusable component. The footer has links to references used to create the website on the right and on the left are social media links. The footer, like the header has a bold background that is contrasted against the light background of the rest of the page to indicate a visual difference.

In order to visually indicate where in the site the user is, the links in the navigation bar will change text color and will become underlined while on that specific page or section. change for wireframes

Blog Template

  1. 1. Blog title

    The blog title is in bold Questrial to draw attention to the title of the blog

  2. 2. Blog Information

    The skeleton acts as a short description of the site for the user to easily identify the key ideas regarding the blog post. I have also included a circular icon for the name of the author and a timestamp to date the post. This is to make the blog posts seem more realistic, like normal blog posts.

  3. 3. Blog Picture

    An image for the blog in a circular shape to follow the style of the rest of the site. This also breaks up the large amount of text on the blog post to keep the user engaged. In the actual website, I have removed the image because I felt that it was unnecessary

  4. 4. Blog References

    The reference list is contained in a block with a darker background in order to visually differentiate the reference list from the actual blog post

  5. 5. Blog Buttons

    The wireframes depict buttons that can redirect the user to different blog posts.

Data Visualization

  1. 1. Title

    The title is in bolded Questrial for visual hierarchy, reinforcing to the user that the important titles of pages are in the same font and size

  2. 2. Data Visualization

    This image is a placeholder for the data visualization. The image is large and in the center of the page to immediately draw the users attention to it. This is the main subject matter of the page and therefore must be the first thing that the user experiences

  3. 3. Description

    To the left of the actual data visualization is the description of the piece to give the user more information. This is secondary information and therefore is set to the side to allow the users gaze to naturally drift to after the visualization. Providing a description and rationale for the data visualisation ensures that the piece is accessible to users who are not reliant on receiving visual information to understand the site. There is a bolded skeleton text when the references for the piece will be

  4. 4. Button Navigation

    I have added a button navigation to allow the user to quickly navigate to the next data visualization quickly. This in-page navigation is a convention of articles. The bold outline of the button allows for it to stand out against the background of the page for easy visual identification

  5. 5. Post Navigation Drawer

    This is a simple drawer that allows the user to see all the available data visualizations at a glance and provides quick navigations to the other data visualisation posts

Data Art

  1. 1. Nav Bar

    This is a small navigation bar for the user to navigate the post as the data-art posts will be longer pieces, This will use hooks to redirect the user to a particular section of the article with ease. This prevents the user from having to scroll to look for relevant information and provides ease of taskmanagement in order to keep the user engaged

  2. 2. Main Artwork

    This big image will be the main artwork. It is position in the center of the page for easy identification and to establish the visual hierarchy of the page

  3. 3. Artwork Gallery

    This is a gallery for the data-art that will depict zoomed-in portions of the artwork. This feature is both functional and aesthetic. The functionality of this gallery is for the user to see the artwork better and view the piece in great detail. The aesthetic value of this is to replicate a digital gallery and to establish this page as one to view and experience digital art

  4. 4. Overview

    A short description of the artwork. This summary of the piece allows users to option of reading this only instead of the full rationale, which can seem like an overwhelming amount of information to some users. The implementation of this overview therefore reflects flexibility in the site.

  5. 5. Rationale

    The rationale of the artwork follows the same layout as the other blog posts on the site for uniformity and consistency

  6. 6. Next Button

    This navigation button allows the user to rapidly navigate between artworks instead of going back to the main page to navigate. This allows for a more seamless experience of the artwork that simulates the action of wafting through a gallery to new artworks. This ease of navigation is therefore an important feature in order to create the specific experience that I intend for the user.
